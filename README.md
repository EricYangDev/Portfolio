# Eric Yang – Interactive EdTech Portfolio

This portfolio is a polished, demo-first personal website built with plain HTML, CSS, and JavaScript for GitHub Pages.

## Portfolio direction

The site presents Eric Yang as an Educational Technology Developer focused on:

- Interactive EdTech tools
- Serious games and gamified learning systems
- AI-assisted educational workflows
- Game engine QA and usability testing
- UI/UX, level design, and product design systems

## Live demos included

1. **Care Conversation Simulator**  
   Branching health-education dialogue with empathy, clarity, and safety feedback.

2. **Scenario-to-Quiz Builder**  
   Client-side activity generator that turns lesson text into questions and reflection prompts.

3. **Outdoor Quest Generator**  
   Serious-games prototype that creates safe movement-based micro quests.

4. **QA Bug Triage Arena**  
   Interactive bug-finding and defect-report scoring demo.

5. **Level Flow Designer**  
   Clickable grid tool for testing player flow, hazards, rewards, and path readability.

6. **Vector Product Configurator**  
   SVG-based product preview with fabrication-style production notes.

7. **Local Learning Analytics Dashboard**  
   Browser-only progress tracking with badges stored in localStorage.

## How to run locally

Open `index.html` in a browser.

No build step, framework, package manager, or external dependency is required.

## How to deploy

This repository is designed for GitHub Pages.

1. Push the files to the `main` branch.
2. Go to **Settings → Pages**.
3. Set the source to the `main` branch.
4. Open the deployed GitHub Pages URL.

## Notes

- All demos are browser-only prototypes.
- The care-related simulator uses fictional educational scenarios and is not medical advice.
- Progress badges are stored locally in the visitor's browser and are not sent anywhere.
- The `images/` folder can be kept for future screenshots, thumbnails, or Open Graph images.
